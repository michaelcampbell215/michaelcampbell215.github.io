Engineering Code Samples

01. Architecture Formulation (Star Schema)

Establishing a 15M+ row Fact Table optimized with foreign keys linked to strictly defined dimensions. 

-- =========================================================================
-- Create Destination Table (Fact Table)
-- =========================================================================
DROP TABLE IF EXISTS general_payments;
CREATE TABLE general_payments (
    payment_id BIGINT AUTO_INCREMENT PRIMARY KEY,
    record_id VARCHAR(50),
    payment_date DATE,
    amount_usd DECIMAL(15, 2),
    number_of_payments INT,
    
    -- Foreign Keys to Dimensions
    payment_nature VARCHAR(200),
    recipient_type VARCHAR(50),
    recipient_specialty VARCHAR(500), 
    payer_name VARCHAR(150),
    
    etl_created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    UNIQUE INDEX idx_record_id (record_id)
);

Result: Created an optimized central fact table, ensuring that the heavy 15 million rows are kept lean by offloading strings to dimension tables.

02. ETL Pipeline (Stored Procedure)

Leveraging INSERT IGNORE batched stored procedures to cleanly populate dimensions and handle missing foreign keys automatically. 

-- Date Dimension
SELECT 'Populating DimDate...' AS Status;
INSERT IGNORE INTO dim_date (date_key, full_date, year, month_name, day_of_week, is_weekend)
SELECT 
    CAST(DATE_FORMAT(date_val, '%Y%m%d') AS UNSIGNED),
    date_val,
    YEAR(date_val),
    MONTHNAME(date_val),
    DAYNAME(date_val),
    CASE WHEN DAYOFWEEK(date_val) IN (1, 7) THEN 1 ELSE 0 END
...

-- Recipient Dimension
SELECT 'Populating DimRecipient...' AS Status;
INSERT IGNORE INTO dim_recipient (
    recipient_id, recipient_name, recipient_type, specialty, city, state, zip, lat, lng
)
SELECT 
    COALESCE(NULLIF(recipient_profile_id, ''), teaching_hospital_ccn),
    MAX(CASE 
        WHEN recipient_type LIKE '%Hospital%' THEN teaching_hospital_name 
        ...
FROM general_payments g
LEFT JOIN ref_zip_city z ON LEFT(g.recipient_zip, 5) = z.zip_code
GROUP BY COALESCE(NULLIF(recipient_profile_id, ''), teaching_hospital_ccn);

Result: Successfully processed messy data fields and missing geography data iteratively by performing dimensional build-outs during pipeline execution.

Data Engineering & Architecture

01

Data Quality as a Bottleneck

The raw compliance dataset of 15 million rows was unsuited for operational reporting. By engineering an overarching Star Schema with a central fact table and five dimension tables, query performance for large executive dashboards shifted from minutes to sub-seconds.

02

The "Hidden Inventory" Discovery

Data buried in secondary columns mapped up to 5 distinct products into single rows. Leveraging a UNION ALL SQL unpivoting algorithm extracted "hidden" inventory volumes that expanded the dataset—revealing legacy supply chain models had been artificially lowering physical product capacity requirements.

03

Spatial Backend Dominance

Implemented MySQL's ST_Distance_Sphere to natively measure distances from mapped coordinates to national logistics hubs. Re-routing this math from the front-end BI layer to deeply optimized Backend procedures provided stable execution scales.
