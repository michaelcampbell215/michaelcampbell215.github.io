Query Showcase

01. Temporal Throughput Analysis

Identifying peak demand windows and operational bottleneck hours using cron-style SQL ordering. 

SELECT
    DAYNAME(order_date)       AS day_of_week,
    HOUR(order_time)          AS hour_of_day,
    COUNT(DISTINCT order_id)  AS number_of_orders
FROM fact_orders
GROUP BY
    DAYOFWEEK(order_date), day_of_week, hour_of_day
ORDER BY
    DAYOFWEEK(order_date), number_of_orders DESC;

Result: Lunch windows on Mon/Fri/Sun are critical throughput peaks. Recommended targeted capacity allocation for these specific windows to maintain service SLAs — the same demand signal engineering used in distribution center staffing and carrier scheduling.

02. GMROI Matrix Classification (CTE + NTILE)

Classifying all SKUs into Stars, Puzzles, Workhorses, and Duds using NTILE quartile tiering — no manual spreadsheet categorization required. 

WITH total_order_revenue AS (
    SELECT
        m.item_name,
        COUNT(o.order_id)  AS total_orders,
        SUM(m.price)       AS total_revenue
    FROM dim_menu_items m
    INNER JOIN fact_orders o ON m.menu_item_id = o.item_id
    GROUP BY m.item_name
),
ItemRanks AS (
    SELECT
        item_name, total_orders, total_revenue,
        NTILE(2) OVER (ORDER BY total_orders DESC)  AS order_ntile,
        NTILE(2) OVER (ORDER BY total_revenue DESC) AS revenue_ntile
    FROM total_order_revenue
)
SELECT
    item_name, total_orders, total_revenue,
    CASE
        WHEN order_ntile = 1 AND revenue_ntile = 1 THEN 'Star'
        WHEN order_ntile = 1 AND revenue_ntile = 2 THEN 'Workhorse'
        WHEN order_ntile = 2 AND revenue_ntile = 1 THEN 'Puzzle'
        WHEN order_ntile = 2 AND revenue_ntile = 2 THEN 'Dud'
    END AS item_category
FROM ItemRanks;

Result: The top-volume SKU was classified as a Workhorse — consuming 25% of peak-hour capacity while delivering minimal net profit. The same Star/Workhorse/Puzzle/Dud framework applies directly to pharmaceutical formulary rationalization and medical supply catalog management.

Building the Analytical Pipeline

01

Star Schema Foundation

Constructed a dimensional model joining operational fact tables with product dimension items to create a reliable analytical layer for complex SQL queries. This architecture prevents the aggregation errors that flat-file volume reports introduce when COGS and margin data are mixed with transactional counts.

02

Market Basket Analysis (Self-Joins)

Executed self-joins on the fact_orders table to surface cross-category purchasing behavior — discovering that customers were naturally pairing specific product categories at a 49% rate that the business was not monetizing. The same self-join affinity pattern is used in pharmaceutical co-prescription analysis and medical supply procurement bundling.

03

Automated CASE Classification

Engineered CASE expressions to drive automated SKU categorization — replacing manual spreadsheet guesswork with a live SQL view that reclassifies every item on each data refresh. The logic is stateless and parameterizable, making it portable to any inventory catalog regardless of domain.
