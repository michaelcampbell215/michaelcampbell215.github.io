Decision Engine

The core logic compares the Cost of Passenger Protection against the Cost of Holding (delay + crew risk + gate conflict) to make a financially-optimal HOLD or PUSH decision per flight. A Regulatory Guardrail overrides the cost comparison when FAA Part 117 duty limits are threatened — regulatory compliance is a hard constraint, not a variable.

analyze_bottleneck_impact.py 
# COST MODELING: NETWORK RISK (DELAY + CONNECTIONS + CREW)
def calculate_network_risk(mins, onboard_pax, crew_risk, fc):
    """
    Aggregates all network costs for a hold.
    Non-linear MCT Cliff: delay costs spike at 15 and 30 min marks.
    """
    base_delay_cost = mins * fc['cost_of_delay_per_min']

    # 1. The MCT Cliff (Connection Risk)
    multiplier = 1.0
    if 15 < mins <= 30: multiplier = 2.5
    elif mins > 30:     multiplier = 5.0

    connection_penalty = (onboard_pax * fc['missed_connection_penalty_per_pax']) * (mins / 60)

    return base_delay_cost + (connection_penalty * multiplier) + crew_risk


# REGULATORY GUARDRAIL: FAA Part 117 Duty Limit Check
def calculate_crew_timeout_risk(hold_time, crew_data, fc):
    remaining_buffer = crew_data['duty_mins_remaining'] - hold_time

    if remaining_buffer <= 10:   # Red Zone — Force Push
        total_impact = fc['cost_of_crew_timeout'] + \
            (crew_data['downstream_legs'] * fc['downstream_cancellation_buffer'])
        return total_impact
    elif remaining_buffer <= 20: # Yellow Zone — Sub-out risk
        return 5000

    return 0  # Green Zone: Crew is safe


# DECISION LOOP (per flight)
# if protect_cost > total_hold_cost:  → HOLD 15M (absorb delay, save pax)
# elif duty_mins_remaining < 15:      → FORCE PUSH (regulatory override)
# else:                               → PUSH NOW (cheaper to re-accommodate)

Building the Decision Framework

01

Manifest Ingestion & Passenger Classification

Parsed complex JSON gate manifests to distinguish Local No-Shows (bags already loaded, requiring pit search) from Security Stalls (passengers delayed at checkpoints, bags not yet at gate). This classification drives the rework penalty calculation — a critical variable that proves a "Push Now" decision can often yield a longer net delay than a strategic 15-minute hold when high-density cargo pits require a 7-minute AFT search.

02

Non-Linear MCT Cliff & Network Cost Modeling

Developed a non-linear delay cost curve where penalties spike at the 15 and 30-minute marks to model hub misconnection exposure. Integrated gate conflict costs ($180/min tarmac fuel burn for inbound aircraft on occupied gates), crew timeout risk, and downstream cancellation exposure into a single aggregate cost function per flight — treating the hub as a network, not a collection of independent departures.

03

Capacity Recovery & Fill-the-Bin Optimization

Utilized latent cargo capacity created by passenger no-shows to clear a 370-bag backlog. Applied Fill-the-Bin logic constrained by Maximum Zero Fuel Weight (MZFW), forward cargo volume limits, and dispatcher-enforced weight and balance parameters. Marginal fuel cost of clearing the backlog: ~$375. Avoided courier fees: ~$37,500. A 100x return on operational hold time.

Strategic Recommendations

  • • High-yield, long-haul segments require protective holding strategies — re-accommodation options are severely limited and protection costs exceed $1,000 per passenger.
  • • Implement automated Part 117 monitoring via duty_mins_remaining to establish hard-stop departure windows before crew duty limits become a network constraint.

Technical Pivot

  • • Evolved from a linear cost-of-delay calculator into a multi-constraint network model once it became clear that pushing on-time creates hidden downstream costs a linear model cannot capture.
  • • Incorporated PPBM logic to account for mandatory baggage-offload delays and the 7-minute AFT search latency — proving intuition fails in high-density cargo configurations.
